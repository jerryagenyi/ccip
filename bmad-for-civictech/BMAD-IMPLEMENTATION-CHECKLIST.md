# BMAD Implementation Checklist

**Step-by-step checklist for implementing BMAD in your civictech organization.**

---

## Phase 1: Foundation (Weeks 1-2)

### Setup & Configuration
- [ ] Review BMAD documentation and strategy documents
- [ ] Verify BMAD installation and configuration
- [ ] Set up output folders (`docs/` structure)
- [ ] Configure BMAD agents for your needs

### Knowledge Base Setup
- [ ] Create `.bmad/bmm/data/civictech-context.md`
  - [ ] Domain terminology
  - [ ] Common patterns
  - [ ] Best practices
  - [ ] Constraints and considerations
- [ ] Create `.bmad/bmm/data/public-health-context.md`
  - [ ] Public health domain knowledge
  - [ ] Crisis communication patterns
  - [ ] Health system integration points
- [ ] Create `.bmad/bmm/data/nonprofit-context.md`
  - [ ] Nonprofit operations
  - [ ] Grant management
  - [ ] Stakeholder management
  - [ ] Impact measurement
- [ ] Create `.bmad/bmm/data/organizational-knowledge.md`
  - [ ] Your organization's mission, values, goals
  - [ ] Key stakeholders, partners, funders
  - [ ] Current systems, processes, constraints
  - [ ] Success metrics, KPIs

### Initial Workflows
- [ ] Run `document-project` workflow (if not done)
- [ ] Run first `research` workflow (test with a real research question)
- [ ] Run first `brainstorm-project` workflow (test with a strategic question)
- [ ] Document learnings and refine approach

**Deliverables:**
- ✅ Domain context files created
- ✅ Organizational knowledge documented
- ✅ First research outputs generated
- ✅ Initial workflow experience documented

---

## Phase 2: Strategic Operations (Weeks 3-6)

### Research Operations
- [ ] Set up regular research schedule (weekly/monthly)
- [ ] Use Research Agent for:
  - [ ] Market intelligence (ongoing)
  - [ ] Policy/regulatory monitoring
  - [ ] Competitive analysis
  - [ ] Funding opportunities
- [ ] Create research output templates
- [ ] Establish research knowledge repository

### Strategic Analysis
- [ ] Use Analyst Agent for:
  - [ ] Opportunity analysis (new markets, features, partnerships)
  - [ ] Risk assessment (technical, operational, strategic)
  - [ ] Resource planning (budget, staffing, timelines)
- [ ] Use `brainstorm-project` workflow for:
  - [ ] Strategic planning sessions
  - [ ] Problem-solving sessions
  - [ ] Innovation sessions
- [ ] Create analysis output templates

### Product Planning
- [ ] Use PM Agent for:
  - [ ] Product briefs (new products, features, programs)
  - [ ] PRDs (detailed requirements)
  - [ ] Roadmap planning
- [ ] Use Architect Agent for:
  - [ ] Technical strategy
  - [ ] System architecture
  - [ ] Technology selection
- [ ] Use `create-epics-stories` for:
  - [ ] Breaking down initiatives into work
  - [ ] Prioritization
  - [ ] Resource allocation

**Deliverables:**
- ✅ Research operations established
- ✅ Strategic analysis processes in place
- ✅ Product planning workflows integrated
- ✅ First strategic outputs generated
