## MEASURING DAILY SUCCESS

### DAILY KPI DASHBOARD
```bash
# Generate Daily Report
/bmad:bmm:agents:analyst "daily performance metrics and insights"

Key Metrics:
- Decision Speed: Average time from intelligence to decision
- Task Completion: % of daily tasks completed
- Quality Score: Code quality, test coverage, bug count
- Intelligence Generated: Number of actionable insights
- Learning Captured: Knowledge base updates and improvements
- Stakeholder Engagement: Communications and responses
```

### REAL-TIME ALERTS
```bash
# Performance Alerts
- Decision latency > 2 hours → PM Agent notification
- Quality score drop → QA Agent intervention
- Task completion < 80% → Dev Agent review
- No new intelligence → Research Agent investigation

# Opportunity Alerts
- New market trends → Research Agent analysis
- Competitor movements → Analyst Agent assessment
- Partnership opportunities → PM Agent evaluation
- Funding announcements → Research Agent deep dive
```

## BEST PRACTICES

### PROMPT ENGINEERING
- Be specific and provide context
- Include relevant data and constraints
- Define clear success criteria
- Use consistent formats and structures

### AGENT COLLABORATION
- Clear handoffs between agents
- Shared context and information
- Defined responsibilities and boundaries
- Regular communication and updates

### KNOWLEDGE MANAGEMENT
- Document all decisions and rationale
- Capture learnings and insights
- Maintain current context information
- Regular knowledge base updates

### QUALITY ASSURANCE
- Multiple agent reviews for critical work
- Automated quality checks and gates
- Continuous testing and validation
- Regular quality assessments and improvements

## TROUBLESHOOTING

### COMMON ISSUES

**Agent Responses Not Relevant**
- Check context files are up to date
- Refine prompts with specific requirements
- Provide examples of desired outputs
- Adjust agent personality settings

**Workflow Bottlenecks**
- Identify specific steps causing delays
- Optimize agent communication
- Automate manual interventions
- Consider parallel execution

**Quality Inconsistencies**
- Strengthen QA agent involvement
- Implement additional review steps
- Define clearer quality criteria
- Use automated quality checks

**Team Adoption Issues**
- Provide additional training and support
- Demonstrate clear value and ROI
- Start with simpler workflows
- Address specific concerns and barriers
